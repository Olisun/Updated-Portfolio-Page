GifTastic

GifTastic

Our project was to create an app that would utilize what we learned in class about jQuery AJAX calls and to query the Giphy API database to get gifs. We have to create topic buttons which were appended to the website. The user would click on a topic button and the website would display 10 gifs from that topic from the API. When the user clicks on another topic button, 10 new gifs would appear and the previous 10 would be replaced. Additionally, there was an input form where the user could enter in a new topic and a new button would appear. When clicked, the same process described above would repeat. Finally, we had to code a feature where the user could click on the gif image to get it to play and then click on the image again to get the gif to stop. This last step proved to be my biggest challenge.

I solved this challenge by applying adding data attributes to the images returned from the API object, another topic we covered in class. This project was probably the best example of me applying the techniques and methods learned in class to make the app work.

Train-Schedules

Train-Schedules

Utilizing what we learned with Firebase’s Realtime Database, we were tasked with building a mock train schedule app where the user would fill out input fields on a form and the app would capture that data and send it up to the Firebase cloud. The data included the name of the train line; the destination; the train departure frequency and what time the first train leaves for the day. Taking this data, we were tasked with calculating when the next train would arrive and the append a schedule back down to the DOM.

This project required not only Firebase database methods, but also Moment.js. The task to write code to accurately predict the train schedule and to format the times with Moment was a great learning expeience. Additionally, I ran into some challenges appending the schedule back down to the table on the website.

My solution to the append issue was pretty simple but it proved to me that my problem solving skills are improving. I created a specific ID in the body section of the table and it worked. Learning Moment.js on the fly was a good practice is utilizing documentation, something that is standard practice in this field and something that I’m still learning to master.

Martin (Blue Kitty) will take you to my Github repository for this project. (Taka) Red Kitty will take you to the deployed site on Github pages.

Trivia Game

Trivia Game

Our project was to create a multiple choice and or true-false game using where the user must answer a set of questions within a given amount of time. If the time runs out, the quiz automatically stops. If the user finishes before time is up, she is given the option to see her score. The game must keep track of answers guessed correctly; answers guess incorrectly as well as unanswered questions.

I chose radio buttons for the HTML DOM elements. Not only did this work stylistically but you can also grab the input values from each radio button using javascript.

This proved to be a challenging assignment for me. Initially, I coded all my logic around each radio button answer. I ran into a major bug where if the user answered a question but then changed her mind, both marked answers were counted.

My solution was to refactor the code around each set of questions rather than each individual radio button answer. Doing it this way, I was able to implement the if / else logic accurately and if the user marked one answer but changed it to another answer, the correct score was calculated.
